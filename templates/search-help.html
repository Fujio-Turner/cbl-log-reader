Search & Filter Guide

← Back to Log Reader

This tool helps you debug Couchbase Lite issues — slowness, errors, sync failures, and unexpected behavior. Search through log entries, filter by type and time range, then drill into the raw data to see what happened before and after a problem.

Search Syntax

Type your search in the search bar at the top of the sidebar. By default, all terms are ANDed together — every term must appear in the log line.

What you type What it does When to use it
push error Both terms required AND Find log lines mentioning both "push" and "error"
push OR error Either term matches OR Broaden search when you're not sure which term appears
-healthcheck Exclude term NOT Remove noisy log lines from results
NOT healthcheck Exclude term (same as -) Alternative negation syntax
"connection refused" Exact phrase match Find a specific error message or sequence of words
retry* Prefix match (retry, retrying, retries…) Match word variations without typing each one
error or errors Shortcut: filters for error=true docs Quickly find all error log entries
Field-Specific Filters

Target a specific field instead of searching all log text. Use the format field:value.

Field Type Example Notes
type text type:Sync Full log type (e.g. Sync:Active)
mainType text mainType:BLIP Top-level type only (e.g. Sync, BLIP)
processId or pid number pid:1047 Must be numeric — wildcards not allowed
error boolean error:true Accepts: true/false, yes/no, 1/0
rawLog text rawLog:timeout The raw log line content (default search field)
dt datetime — ISO timestamp — use the date pickers instead
Tip: Field names are case-insensitive in search — processid:1047, ProcessId:1047, and pid:1047 all work.
Combining Searches

You can combine multiple terms, field filters, and operators in a single search.

Example What it finds
type:Sync AND error:true Sync-type log entries that are errors
push AND error -healthcheck Logs with "push" and "error", excluding "healthcheck"
pid:1047 "connection refused" Exact phrase in a specific process
timeout OR "timed out" Either the word "timeout" or the phrase "timed out"
retry* -healthcheck -ping Anything starting with "retry", excluding noise
Sidebar Controls

Type Mode

General Type — Groups logs by top-level type (e.g. Sync, BLIP, DB). Best for getting an overview.

Specific Type — Uses the full type (e.g. Sync:Active, BLIP:WebSocket). Use when you need to drill into a specific subsystem.

Type Select

Multi-select list of all log types found in the data. Select one or more to filter the chart and raw data. Use Select All / Unselect All to toggle quickly. The special Error entry filters for logs with error=true.

Grouping Mode

By Second — Chart shows log counts per second. Best for short time ranges or finding precise spikes.

By Minute — Aggregates per minute. Better for longer time ranges to see overall patterns.

Date Range

Set start and end timestamps to narrow your search window. The date pickers include seconds for precision. Dates auto-populate from the earliest and latest log entries in the data.

Result Limit

Controls how many raw log rows are returned. Default is 500. Higher limits show more data but take longer to load. None removes the limit entirely — use with caution on large datasets.

Chart Scale (Linear / Logarithmic)

Linear — Standard scale. Good when log volume is fairly even.

Logarithmic — Compressed scale. Use when a few spikes make it hard to see smaller counts.

Debugging Workflow

Common patterns for finding and investigating Couchbase Lite issues:

  • Start broad — Load all logs for the time range. Look at the chart for spikes or unusual patterns.
  • Find errors — Type error in search or select the Error type to see all error entries. Note the timestamps.
  • Narrow the time window — Set the date range to just before and after the error spike. Switch to "By Second" grouping for precision.
  • Filter by type — Select only Sync or BLIP types to focus on replication issues. Use Specific Type mode for more detail.
  • Examine raw logs — Scroll through the raw data table. Use Stakes to bookmark rows you want to compare. Look at what happened before and after the error.
  • Isolate by process — Use pid:1047 to see only logs from a specific process, filtering out noise from other processes.

Common Searches for CBL Debugging

Issue Search
Sync errors type:Sync error:true
Connection problems "connection refused" OR timeout OR "timed out"
Replication failures type:Sync push error
WebSocket issues mainType:BLIP error:true
Slow operations slow* OR timeout*
Everything except noise -healthcheck -ping -heartbeat
Important Notes
Full-text search, not substring search. The search engine tokenizes text into words. Searching for time will not match timeout — use time* for prefix matching.
Leading wildcards (*foo) are not supported. Use prefix wildcards only: foo*.
processId is numeric. You cannot use wildcards on process IDs. Use exact values like pid:1047.
Negative-only queries need a base. A search like -healthcheck alone needs date/type filters to have something to exclude from.
